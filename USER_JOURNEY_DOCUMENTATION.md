## Client Journey

### 1. Registration & Onboarding

**Step 1: Account Creation**
- Client visits the platform and clicks "Sign Up"
- Fills in registration form:
  - Email address
  - Username
  - Password
  - Full name
  - Phone number (optional)
- Verifies email address via activation link
- Completes profile setup

**Step 2: Profile Completion**
- Uploads profile picture (optional)
- Sets notification preferences
- Configures payment methods
- Reviews terms and conditions

**Step 3: First Order**
- Client navigates to "Place Order" page
- Fills in order details:
  - Topic/title
  - Paper type (essay, research paper, etc.)
  - Academic level
  - Number of pages
  - Deadline
  - Subject area
  - Formatting style (APA, MLA, etc.)
  - Detailed instructions

### 2. Order Placement with Style References (New Feature)

**Step 1: Create Order**
- Client fills in standard order form
- **NEW**: Optional section for "Style Reference Files"
  - Client can upload previous papers they've written
  - Client can upload papers with instructor feedback
  - Client can upload style guides or sample work
  - Multiple files can be uploaded
  - Each file can have a description

**Step 2: Upload Style References**
- Client clicks "Upload Style Reference" button
- Selects file type:
  - Previous Paper
  - Instructor Feedback
  - Style Guide
  - Sample Work
- Uploads file(s) (PDF, DOCX, etc.)
- Adds optional description for each file
- Can toggle visibility to writer (default: visible)
- Can upload multiple style references

**Step 3: Review & Submit**
- Client reviews order details
- Reviews uploaded style references
- Applies discount code (if available)
- Selects preferred writer (optional)
- Submits order

**Step 4: Payment**
- Client is redirected to payment page
- Selects payment method:
  - Credit/Debit card
  - PayPal
  - Wallet balance
- Completes payment
- Receives confirmation email

### 3. Order Management

**Step 1: Order Tracking**
- Client views order dashboard
- Sees order status:
  - Created → Unpaid
  - Paid → Available/Assigned
  - In Progress
  - Under Editing
  - Completed
  - Approved
  - Revision Requested

**Step 2: Communication**
- Client can message assigned writer
- Receives notifications for:
  - Order assignment
  - Progress updates
  - Completion
  - Revision requests

**Step 3: Review & Approval**
- When order is completed:
  - Client receives notification
  - Reviews completed work
  - Can request revisions (if within free revision window)
  - Can approve order
  - Can rate writer

**Step 4: Download Final Paper**
- After payment confirmation:
  - Client can download final paper
  - Receives all deliverables
  - Can access order files

### 4. Style Reference Management

**Step 1: View Uploaded References**
- Client can view all style references uploaded for an order
- Can see:
  - File name
  - File type
  - Upload date
  - Visibility status

**Step 2: Manage References**
- Client can:
  - Toggle visibility to writer
  - Add descriptions
  - Delete references (before order is assigned)
  - Upload additional references (before order starts)

**Step 3: Privacy Control**
- Client controls whether writer can see each reference
- Can hide sensitive information
- Can make references visible only after order assignment

## Writer Journey

### 1. Registration & Profile Setup

**Step 1: Account Creation**
- Writer applies to join platform
- Fills application form:
  - Personal information
  - Educational background
  - Writing experience
  - Subject expertise
  - Sample work (optional)

**Step 2: Profile Verification**
- Admin reviews application
- Writer receives approval/rejection
- If approved, writer completes profile:
  - Uploads resume/CV
  - Sets availability
  - Configures payment preferences
  - Sets notification preferences

**Step 3: Writer Level Assignment**
- Admin assigns writer level:
  - Standard
  - Premium
  - Elite
- Writer can see level benefits and requirements

### 2. Order Discovery & Selection

**Step 1: Browse Available Orders**
- Writer navigates to "Order Queue"
- Views available orders:
  - Filter by subject
  - Filter by deadline
  - Filter by price
  - Filter by academic level

**Step 2: Order Details Review**
- Writer clicks on order to view details
- Sees:
  - Topic and instructions
  - Requirements (pages, formatting, etc.)
  - Deadline
  - Price and compensation
  - **NEW**: Style reference files (if uploaded and visible)
  - Client rating (if available)

**Step 3: Access Style References**
- Writer can view style reference files:
  - Previous papers by client
  - Instructor feedback
  - Style guides
  - Sample work
- Can download files to study writing style
- Can use references to match client's writing style

**Step 4: Take or Request Order**
- Writer can:
  - Take order directly (if capacity allows)
  - Request order (if preferred writer)
  - Decline order

### 3. Order Execution

**Step 1: Order Assignment**
- Writer receives notification of assignment
- Reviews order details thoroughly
- **NEW**: Downloads and studies style reference files
- Analyzes:
  - Writing tone and style
  - Formatting preferences
  - Common feedback themes
  - Academic level expectations

**Step 2: Work Planning**
- Writer creates work plan
- Sets internal milestones
- Estimates completion time
- Starts working on order

**Step 3: Progress Updates**
- Writer can update progress percentage
- Can send messages to client
- Can request clarifications
- Can upload draft files (if draft requests enabled)

**Step 4: Style Matching**
- Writer uses style references to:
  - Match writing tone
  - Follow formatting patterns
  - Address common feedback points
  - Maintain consistency with client's previous work

**Step 5: Completion**
- Writer completes order
- Reviews work against:
  - Order requirements
  - Style references
  - Client instructions
- Uploads final paper
- Submits order for review

### 4. Order Revision (If Requested)

**Step 1: Revision Request**
- Writer receives revision request
- Reviews client feedback
- Accesses style references again (if needed)
- Makes requested changes

**Step 2: Revision Submission**
- Writer uploads revised version
- Marks revision as complete
- Waits for client approval

### 5. Payment & Earnings

**Step 1: Order Approval**
- Client approves order
- Writer receives payment notification
- Earnings added to writer wallet

**Step 2: Payment Processing**
- Writer can:
  - View earnings dashboard
  - Request withdrawal
  - Set payment schedule preferences
  - View payment history

## Style Reference Feature Workflow

### Complete Workflow Diagram

```
Client Side:
1. Create Order
   ↓
2. Upload Style References (Optional)
   - Select file type
   - Upload file(s)
   - Add description
   - Set visibility
   ↓
3. Submit Order
   ↓
4. Complete Payment
   ↓
5. Order Assigned to Writer

Writer Side:
1. View Available Orders
   ↓
2. Review Order Details
   - Read instructions
   - View style references (if visible)
   ↓
3. Take/Request Order
   ↓
4. Download Style References
   - Study writing style
   - Analyze formatting
   - Review feedback patterns
   ↓
5. Complete Order
   - Match style from references
   - Follow formatting patterns
   - Address common feedback themes
   ↓
6. Submit Order
```

### Style Reference Types

1. **Previous Paper**
   - Client's own previous work
   - Shows writing style and quality
   - Helps match tone and structure

2. **Instructor Feedback**
   - Papers with professor comments
   - Shows common feedback themes
   - Helps avoid previous mistakes

3. **Style Guide**
   - Formatting guidelines
   - Citation requirements
   - Academic standards

4. **Sample Work**
   - Example papers
   - Reference materials
   - Quality benchmarks

### Privacy & Access Control

- **Client Controls:**
  - Can upload multiple references
  - Can toggle visibility per file
  - Can add descriptions
  - Can delete before assignment

- **Writer Access:**
  - Can view visible references
  - Can download files
  - Can access throughout order lifecycle
  - Cannot see hidden references

- **Admin/Support:**
  - Full access to all references
  - Can manage files
  - Can troubleshoot access issues

## Order Lifecycle

### Status Flow

```
CREATED (Unpaid)
    ↓
PAID
    ↓
AVAILABLE (Open to writers)
    ↓
ASSIGNED / IN_PROGRESS
    ↓
UNDER_EDITING (Optional)
    ↓
COMPLETED
    ↓
APPROVED / REVISION_REQUESTED
    ↓
ARCHIVED
```

### Key Milestones

1. **Order Creation**
   - Client submits order
   - Style references uploaded (optional)
   - Order status: CREATED

2. **Payment**
   - Client completes payment
   - Order status: PAID → AVAILABLE

3. **Assignment**
   - Writer takes/requests order
   - Writer accesses style references
   - Order status: ASSIGNED → IN_PROGRESS

4. **Completion**
   - Writer submits work
   - Order status: COMPLETED

5. **Review**
   - Client reviews work
   - Can request revision or approve
   - Order status: REVISION_REQUESTED or APPROVED

6. **Finalization**
   - Order approved
   - Files delivered
   - Payment processed
   - Order status: ARCHIVED

## Revision Process

### Revision Request Flow

1. **Client Reviews Work**
   - Downloads completed paper
   - Reviews against requirements
   - Checks style consistency

2. **Revision Request**
   - Client submits revision request
   - Provides specific feedback
   - References style guide if needed

3. **Writer Revises**
   - Writer receives notification
   - Reviews feedback
   - Accesses style references again
   - Makes changes

4. **Revision Submission**
   - Writer uploads revised version
   - Client reviews again
   - Process repeats if needed

### Free Revision Window

- Revisions are free within specified time period
- Typically 7-14 days after completion
- After window, revisions may incur fees
- Unlimited revisions during free window

## Best Practices

### For Clients

1. **Style Reference Upload**
   - Upload clear, relevant examples
   - Include instructor feedback when available
   - Provide context in descriptions
   - Upload before order assignment for best results

2. **Order Instructions**
   - Be specific and detailed
   - Reference style guides
   - Mention important requirements
   - Provide examples when helpful

3. **Communication**
   - Respond promptly to writer messages
   - Provide clarifications when asked
   - Give constructive feedback

### For Writers

1. **Style Reference Usage**
   - Download and study references early
   - Note common patterns
   - Match tone and formatting
   - Address feedback themes

2. **Order Execution**
   - Plan work before starting
   - Update progress regularly
   - Communicate proactively
   - Meet deadlines

3. **Quality Assurance**
   - Review against requirements
   - Check style consistency
   - Proofread thoroughly
   - Use style references as guide

## Troubleshooting

### Common Issues

1. **Style Reference Upload Fails**
   - Check file size limits
   - Verify file format
   - Check internet connection
   - Contact support if persists

2. **Writer Cannot Access References**
   - Check visibility settings
   - Verify order assignment
   - Contact support for assistance

3. **File Download Issues**
   - Check browser compatibility
   - Verify file permissions
   - Clear browser cache
   - Try different browser

## API Endpoints Reference

### Style Reference Endpoints

- `GET /api/v1/order-files/style-reference-files/` - List style references
- `POST /api/v1/order-files/style-reference-files/` - Upload style reference
- `GET /api/v1/order-files/style-reference-files/{id}/` - Get details
- `GET /api/v1/order-files/style-reference-files/{id}/download/` - Download file
- `POST /api/v1/order-files/style-reference-files/{id}/toggle_visibility/` - Toggle visibility
- `DELETE /api/v1/order-files/style-reference-files/{id}/` - Delete file

### Order Endpoints

- `GET /api/v1/orders/orders/` - List orders
- `POST /api/v1/orders/orders/create/` - Create order
- `GET /api/v1/orders/orders/{id}/` - Get order details (includes style_reference_files)
